jc-club平台采用前后端分离的B/S架构，整体技术选型围绕DDD微服务设计理念展开，具体技术栈如下。

后端技术方面，系统以Spring Boot 3.x作为微服务的基础开发框架，通过Spring Cloud Alibaba提供的Nacos实现服务注册与发现，Gateway实现统一API网关路由，OpenFeign实现声明式服务间远程调用。[2]。在数据持久化层面，采用MySQL 5.7作为主要关系型数据库，使用MyBatis-Plus作为ORM框架，简化数据库操作并提升开发效率。缓存层面，集成Redis实现热点数据缓存（如题目分类树、用户会话Token）和分布式锁功能，显著降低数据库查询压力。[17]搜索引擎层面，引入Elasticsearch对题目数据进行全文索引，支持按题目名称、题干内容、标签等维度的关键词检索，并实现搜索结果的相关性排序。[19]消息队列层面，采用Apache RocketMQ实现微服务间的异步解耦通信，典型应用场景包括题目点赞数据的异步同步、练习记录的异步存储等。[2]在认证与安全方面，系统采用JWT（JSON Web Token）实现无状态的用户认证和会话管理，结合Spring Security实现方法级的权限控制，支持基于RBAC（基于角色的访问控制）模型的权限管理。[18]

前端技术方面，系统采用React 18作为核心UI框架，使用TypeScript进行类型安全的代码编写，以Vite作为前端构建工具，实现快速的开发热更新和优化的生产构建。[3]状态管理采用Redux Toolkit，路由管理采用React Router v6。在UI组件方面，系统使用Ant Design组件库提供统一的界面风格和交互体验，并结合CSS Modules和Less实现模块化的样式管理。前端通过Axios发送HTTP请求与后端API进行数据交互，支持请求拦截器和响应拦截器的统一配置，实现Token自动携带和异常统一处理。[3]

在微服务工程结构方面，jc-club严格遵循DDD的四层架构模型：API层（api）对外暴露Feign接口和DTO定义，供其他微服务调用；应用层（application）负责请求参数校验、DTO与BO之间的转换，以及业务用例的编排；领域层（domain）沉淀核心业务逻辑，包含领域实体（BO）、领域服务和业务规则策略；基础设施层（infra）封装数据库访问（Mapper/DAO）、缓存操作、搜索引擎集成和消息队列收发等技术细节。[7]这种分层设计将业务语义与技术实现彻底解耦，使得各层职责清晰，便于单元测试、功能扩展和团队协作。[4]。

在系统部署层面，平台采用Docker容器化部署方案，各微服务模块打包为独立的Docker镜像，通过Docker Compose进行本地环境的快速搭建，并预留了Kubernetes编排接口以支持后续的集群化部署和弹性伸缩。[9]

整体技术架构以高可用、高性能和高扩展性为核心目标，通过微服务拆分、分布式中间件整合和模块化前端工程，为jc-club平台的稳定运行和持续演进提供了坚实的技术基础。
